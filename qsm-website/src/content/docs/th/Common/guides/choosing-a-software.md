มีซอฟต์แวร์เซิร์ฟเวอร์สองประเภทใน QSM ได้แก่ **พรอกซี (Proxy)**
และ **เซิร์ฟเวอร์เกม**

## เซิร์ฟเวอร์เกม
**เซิร์ฟเวอร์เกม** คือตัวเซิร์ฟเวอร์หลักที่จัดการโลกภายในเกม ข้อมูลผู้เล่น ฯลฯ และเซิร์ฟเวอร์เกมสามารถรองรับ ดาตาแพค (datapack), ม็อด (mods), และ/หรือปลั๊กอินได้

**ม็อด** (Mods ซึ่งย่อมาจาก Modifying หรือ "การดัดแปลง") โดยปกติแล้วคือการดัดแปลงแกนกลางของเกม
และมีความจำเป็นที่ทั้งไคลเอนต์ (ตัวผู้เล่น) และเซิร์ฟเวอร์จะต้องมีมอดหรือการดัดแปลงเดียวกัน
อย่างไรก็ตามไม่ใช่ทุกม็อดที่จะตามเงื่อนไขนี้ เพราะบางม็อดเป็นม็อดสำหรับฝั่งไคลเอนต์หรือฝั่งเซิร์ฟเวอร์เท่านั้น
ตัวอย่างได้แก่

- [Create](https://modrinth.com/mod/create) - เพิ่มบล็อคและกลไกใหม่ ๆ และจำเป็นที่ทั้งสองฝั่งต้องมีม็อดนี้อยู่
- [Sodium](https://modrinth.com/mod/sodium) - ม็อดนี้เป็นม็อดสำหรับฝั่งไคลเอนต์เท่านั้น เนื่องจากม็อดนี้ทำการแก้ไขการเร็นเดอร์ของเกม และเซิร์ฟเวอร์ไม่ต้องทำการเร็นเดอร์อยู่แล้ว
- [Xaero's Minimap](https://modrinth.com/mod/xaeros-minimap) - ม็อดนี้เป็นม็อดสำหรับฝั่งไคลเอนต์เป็นหลัก อย่างไรก็ตามเซิร์ฟเวอร์สามารถติดตั้งม็อดนี้ได้เพื่อฟีเจอร์เพิ่มเติม อย่างไรก็ตามหากติดตั้งบนเซิร์ฟเวอร์แล้วจะเป็นการบังคับให้ทุกผู้เล่นที่เข้าต้องมีม็อดนั้น

**ปลั๊กอิน (Plugins)** สามารถถูกเรียกว่าม็อดฝั่งเซิร์ฟเวอร์ก็ว่าได้ โดยปลั๊กอินจะสามารถแก้ไขตัวเกมได้ แต่ไม่ถึงระดับที่ม็อดสามารถทำได้
โดยปกติแล้วปลั๊กอินจะเพิ่มคำสั่งและเครื่องมือต่าง ๆ เข้าไปในเกม แต่ไม่ใช่การเพิ่มบล็อคใหม่หรือการเปลี่ยนแปลงกลไกของเกมไปอย่างมาก
แต่เซิร์ฟเวอร์ที่มีแค่ปลั๊กอินจะไม่มีการบังคับให้ฝั่งไคลเอนต์ต้องติดตั้งอะไรเพิ่มเติม ดังนั้นผู้ใช้จึงสามารถเปิดเกมเวอร์ชันวานิลลา
(เวอร์ชันที่ไม่มีการดัดแปลง) และเข้าร่วมได้เลย

### Paper
[Paper](https://papermc.io/software/paper) เป็นซอฟต์แวร์เซิร์ฟเวอร์ Minecraft 
ที่โด่งดัง ถูกสร้างขึ้นมาจาก Spigot และถูกออกแบบเพื่อเพิ่มประสิทธิภาพและให้ฟีเจอร์และ API ขั้นสูงเพิ่มเติม

ประสิทธิภาพคือเหตุผลหลักที่ Paper คือตัวเลือกค่าเริ่มต้น แม้ว่าคุณจะไม่ใช้ปลั๊กอิน Paper 
ยังเป็นทางเลือกที่ดีอยู่เหมือนเดิม

คุณสามารถอ่านเอกสารคู่มือของ Paper ได้ที่ https://docs.papermc.io/paper

### Purpur
[Purpur](https://purpurmc.org/) เป็นซอฟต์แวร์เซิร์ฟเวอร์ที่มีรากฐานมาจาก Paper 
โดย Purpur คือซอฟต์แวร์ที่ออกแบบมาเพื่อเพิ่มตัวเลือกในการปรับแต่งเพิ่มเติม และฟีเจอร์ใหม่ ๆ ที่น่าสนุกและน่าตื่นเต้น
(เขาว่ามาอย่างนั้น)

อ่านเอกสารคู่มือของ Purpur ได้ที่ https://purpurmc.org/docs/purpur/

### วานิลลา
[เซิร์ฟเวอร์วานิลลา](https://www.minecraft.net/en-us/download/server)
เพียงแค่หมายถึงซอฟต์แวร์เซิร์ฟเวอร์อย่างเป็นทางการที่ไม่ถูกดัดแปลงจาก Minecraft

คุณสามารถอ่านหน้าวิกิที่พูดถึงการตั้งค่าเซิร์ฟเวอร์ Minecraft ได้[ที่นี่](https://minecraft.wiki/w/Tutorial:Setting_up_a_Java_Edition_server)
อย่างไรก็ตามคำแนะนำในหน้านั้นอาจใช้กับ QSM ไม่ได้ทั้งหมด

### Fabric
[Fabric](https://fabricmc.net/) คือตัวโหลดม็อด Minecraft ที่แยกส่วนได้และบางเบา

เมื่อสร้างเซิร์ฟเวอร์ Fabric QSM จะทำการเลือกตัวติดตั้งเวอร์ชันล่าสุดเสมอ

คุณสามารถอ่านเอกสารคู่มือของ Fabric ได้ที่ https://docs.fabricmc.net/

### Forge

Forge เป็น API สำหรับม็อดที่ถูกใช้งานอย่างแพร่หลาย โดยคุณสามารถอ่านเอกสารคู่มือของ 
Forge ได้ที่ https://docs.minecraftforge.net/en/1.21.x/

### NeoForge
[NeoForge](https://neoforged.net/) เป็น API สำหรับม็อดที่ฟรี โอเพนซอร์ส และมุ่งเน้นชุมชน

คุณสามารถอ่านเอกสารสำหรับการสร้างม็อดด้วย NeoForge ได้ที่ 
https://docs.neoforged.net/docs/gettingstarted/

## พรอกซี
**พรอกซี (Proxy)** คือซอฟต์แวร์ที่ทำหน้าที่ในการนำทางเกม Minecraft ไปยังเซิร์ฟเวอร์ต่าง ๆ 
ซึ่งจะถูกใช้ในการตั้งค่าเครือข่ายที่มีเซิร์ฟเวอร์เกมย่อยหลายเซิร์ฟเวอร์

### Velocity
[Velocity](https://papermc.io/software/velocity) คือซอฟต์แวร์พรอกซีทันสมัยและประสิทธิภาพสูงจากทีม
PaperMC ซึ่งถูกออกแบบมาเพื่อประสิทธิภาพและสเถียรภาพโดยเฉพาะ และเป็นซอฟต์แวร์ทางเลือกของ 
Waterfall และ BungeeCord ที่มีสิ่งแวดล้อมปลั๊กอินของตนเอง

คุณสามารถอ่านเอกสารคู่มือ Velocity ได้ที่ https://docs.papermc.io/velocity
